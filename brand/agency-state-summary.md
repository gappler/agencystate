## The Problem We Solve

This isn't a software problem. It's a context problem.

Every team has knowledge, judgment, and institutional memory that lives in people's heads — brand voice that one person carries, competitive instincts that never get documented, strategic context that's shared in hallway conversations. There has never been a reliable way to capture that context, structure it, and put it to work.

AI changes that — but only if it has your context. Generic AI produces generic output. AI with your brand voice, your data, your audience definitions, your competitive landscape produces work your team can actually use.

The most important work your team does — writing briefs, adapting messaging, building plans, analyzing markets — has never had a partner. Now it does.

## Methodology: Foundation First, Tools Second

Every engagement and every training cohort follows the same system:

1. **Capture the context** — define the business (brand voice, audience segments, product details, competitive landscape, team structure). Pull what lives in people's heads into structured form. This is the foundation that makes everything downstream specific and useful.

2. **Build sequentially** — each tool feeds the next. The sequence matters. A brand configuration governs a content engine. Research produces a brief. The brief feeds the content. Remove any layer and the output loses fidelity.

3. **Keep humans in the loop** — AI produces drafts at every stage. Humans make decisions at every gate. The system makes these checkpoints visible and concrete, not implied.

4. **Produce practical outputs** — the value is not dashboards and presentations. It's the text file that feeds Salesforce, the CSV that imports to Smartsheet, the brief that feeds Jasper, the content matrix in spreadsheet format. Tools that capture context through forms, interviews, and structured inputs, and produce outputs existing systems can actually use.

**The workflow:** Claude (chat) for thinking — strategic planning, research, deciding what to build and why. Claude Code (terminal) for doing — building tools, generating data, creating files, committing code. The two together are the partnership. One helps you figure out *what* to build, the other builds it.

**This methodology is the product differentiator.** Anyone can show someone how to prompt Claude Code. Agency State teaches a system for building with it.

## Key Insights (from building)

- **The build is fast, the understanding is slow.** A motivated person can build a brand definition, generate 12 months of data, and create a working dashboard in under an hour. Speed is the hook. Knowing what to build, verifying what Claude decided, and adapting the approach is the real value.
- **"Build" not "design."** When Claude Code asks about mockups or specs, it shifts into design mode. If you want working tools, say "build." Specs are useful for complex UI decisions but they're not the deliverable.
- **Git discipline is non-negotiable.** Set up git before building anything. Commit after every tool. Learned the hard way.
- **Prework filters and accelerates.** Students who arrive with Claude Code installed and a demo company created can start building immediately. The prework also filters — anyone who can't get through it isn't ready for the intensive.
- **Demos are marketing. Training is the product.** The demos make someone want it. The intensive teaches them to do it. The gap between "impressive demo" and "I can do this myself" is exactly where Agency State lives.
- **Practical outputs over pretty dashboards.** Real client work produces intake forms, CSVs, importable project plans, content matrices, and structured briefs — not just interactive visualizations. The dashboards demo well. The practical outputs get used.
